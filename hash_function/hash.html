Porównywanie funkcji skrótu¶

W tym sprawozdaniu bdziemy porwnywa trzy funkcje skrtu: md5, sha256 oraz sha384. Funkcje te su do generowania krtkiego, staego i unikalnego kodu dla dowolnego cigu danych. S one szeroko stosowane w informatyce, midzy innymi do weryfikacji integralnoci plikw i hase. Celem naszego porwnania bdzie zbadanie, ktra funkcja skrtu zapewnia najwyszy poziom bezpieczestwa i skutecznoci. W tym celu przetestujemy te funkcje na rnych rodzajach danych, takich jak krtkie i dugie cigi znakw oraz pliki o rnych rozmiarach. Wszystkie testy zostan wykonane w jzyku Python z wykorzystaniem biblioteki hashlib.

Tworzenie plików¶

Tworzymy trzy pliki o rozmiarach 1, 5 i 10 megabajtw w katalogu "files". Funkcja create_file suy do generowania plikw o okrelonym rozmiarze, a nastpnie jest wywoywana trzy razy w ptli for z argumentami 1, 5 i 10, aby utworzy trzy pliki o rnych rozmiarach. Dodatkowo w funkcji create_file istnieje opcjonalny argument "amount_1_bin", ktry suy do zdefiniowania, ile znakw o 1 bicie pojawi si w pliku wynikowym.

In [3]:
file_samples = [1, 5, 10]
files_directory = 'files'

def create_file(size_in_mb, directory=files_directory, amount_1_bin=0):
    size_in_bytes = size_in_mb * 1024 * 1024

    if not os.path.exists(directory):
        os.makedirs(directory)

    with open(f'{directory}/{size_in_mb}MB.txt', 'wb') as f:
        f.write(b'\0' * (size_in_bytes - amount_1_bin))
        f.write(b'\1' * amount_1_bin)

for sample in file_samples:
    create_file(sample)

Wyliczanie czasów działania algorytmów¶

Liczymy czas, jaki zajmuje poszczeglnym algorytmom na skrcenie kadego z trzech plikw. Kady pomiar wykonujemy 30 razy i go uredniamy w celu zachowania transparentnoci wynikw.

def calculate_hash_time(samples, algorithm):
    sample_times = []
    for sample in samples:
        with open(f'{files_directory}/{sample}MB.txt', 'rb') as f:
            start = time.time()
            hash_obj = algorithm(f.read())
            end = time.time()

            sample_times.append(end - start)

    return list(zip(samples, sample_times))

def calculate_avg_time(samples, algorithm, acc):
    times = [0] * len(samples)
    for _ in range(acc):
        hash_time = calculate_hash_time(samples, algorithm)
        hash_time_list = [element[1] for element in hash_time]
        times = [a + b for a, b in zip(times, hash_time_list)]

    return [hash_time / acc for hash_time in times]

md5_time = calculate_avg_time(file_samples, hashlib.md5, 30)
sha256_time = calculate_avg_time(file_samples, hashlib.sha256, 30)
sha384_time = calculate_avg_time(file_samples, hashlib.sha384, 30)

Algorytmy SHA-384 i SHA-256 s oparte na tej samej konstrukcji, a rni si jedynie dugoci wyniku (384 bity dla SHA-384 i 256 bitw dla SHA-256). Jednak SHA-384 wykonuje wicej rund oblicze (80 rund) ni SHA-256 (64 rundy), co przekada si na zwikszenie zoonoci obliczeniowej. Dodatkowo SHA-384 korzysta z wikszych wartoci pocztkowych i uywa wikszych kluczy podczas przetwarzania danych wejciowych, co rwnie wpywa na zwikszenie zoonoci obliczeniowej.

def compare_hashes(samples, algorithm):
    hashes = []
    for sample in samples:
        create_file(sample)
        with open(f'{files_directory}/{sample}MB.txt', 'rb') as f:
            hash_obj_0 = algorithm(f.read())
            hash_0 = hash_obj_0.hexdigest()

        create_file(sample, amount_1_bin=1)
        with open(f'{files_directory}/{sample}MB.txt', 'rb') as f:
            hash_obj_1 = algorithm(f.read())
            hash_1 = hash_obj_1.hexdigest()

        hashes.append((sample, hash_0, hash_1))

    return hashes

def hex_to_bin(hex_str):
    hex_num = int(hex_str, 16)
    binary_str = bin(hex_num)[2:].zfill(len(hex_str) * 4)
    return [int(b) for b in binary_str]

def hash_difference_per(data_set_1, data_set_2):
    data_set = zip(hex_to_bin(data_set_1), hex_to_bin(data_set_2))
    difference_counter = sum([1 for x, y in data_set if x != y])

    return (difference_counter / len(hex_to_bin(data_set_1))) * 100

table.add_row(["MD5"] + [f"{round(hash_difference_per(x, y), 4)}%" for _, x, y in md5_hashes])

Analiza kolizji dla algorytmów skrótu¶

Tworzymy losowo 500 i 1000 stringw o rnych dugociach. Nastpnie dla kadego algorytmu skrtu tworzymy hash. Sprawdzamy w zbiorze, ile jest takich hashw, ktre maj takie same pierwsze 12 i 20 bitw. Wyniki prezentujemy w tabeli.

def create_random_hashes(n, algorithm):
    hashes = []
    for _ in range(n):
        rand_string = ''.join(random.choices(string.ascii_uppercase + string.digits, k=random.randint(10, 100)))
        hash_obj = algorithm(rand_string.encode("UTF-8"))
        hashes.append(hash_obj.hexdigest())

    return hashes

def get_n_first_bits_from_hash(n, hash_hex):
    bits = []
    for char in hash_hex:
        bits += [int(b) for b in bin(int(char, 16))[2:].zfill(4)]

    return "".join(map(str, bits[:n]))

def analyze_algorithm(algorithm, first_bits, acc):
    random_hashes = create_random_hashes(acc, algorithm)
    hashes_first_bits = []
    for random_hash in random_hashes:
        hashes_first_bits.append(get_n_first_bits_from_hash(first_bits, random_hash))

    hashes_first_bits_set = set(hashes_first_bits)

    return len(hashes_first_bits) - len(hashes_first_bits_set)

Na podstawie przedstawionych wynikw, mona zauway, e algorytmy skrtu MD5, SHA256 i SHA384 w przypadku pierwszych 12 bitw maj pewne kolizje. Dla 500 hashw MD5 ma 24 powtrzenia, a dla 1000 hashw - 108 powtrze. W przypadku SHA256 jest to odpowiednio 25 i 111 powtrze, a dla SHA384 - 29 i 119 powtrze.

Natomiast dla pierwszych 20 bitw, tylko MD5 ma jedno powtrzenie dla 1000 hashw, pozostae algorytmy nie maj kolizji.

Naley jednak zauway, e 12 i 20 bitw to stosunkowo maa liczba bitw, ktre nie zapewniaj dostatecznego poziomu bezpieczestwa. W praktyce, do bezpiecznego skrtu danych zwykle uywa si co najmniej 128-bitowych funkcji skrtu, takich jak SHA-256 czy SHA-512.

Podsumowujc, wyniki tej analizy wskazuj, e MD5, SHA256 i SHA384 maj pewne kolizje dla maych liczb bitw, ale nie maj one znaczcego wpywu na ich bezpieczestwo w praktyce. Jednak dla wikszego poziomu bezpieczestwa zaleca si uywanie bardziej zaawansowanych funkcji skrtu.
